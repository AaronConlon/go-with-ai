# Knowledge Base

The knowledge base is organized by the same six stages as the learning path. Each stage stores goals, tasks, concepts, code snippets, questions, reviews, and acceptance criteria.

## Modules

| Module | Stage | Focus |
| --- | --- | --- |
| [Stage 1: Startup](/en/notes/stage-1-startup/) | Go module and basics | Environment, commands, project structure, first tests |
| [Stage 2: Networking](/en/notes/stage-2-network/) | HTTP and JSON | HN API, `context`, error boundaries |
| [Stage 3: Concurrency](/en/notes/stage-3-concurrency/) | goroutine and bounded concurrency | `errgroup`, cancellation, concurrency limits |
| [Stage 4: Service](/en/notes/stage-4-service/) | Long-running service | API server, config, logs, health checks |
| [Stage 5: AI Integration](/en/notes/stage-5-ai-integration/) | LLM API calls | Structured output, retry, rate limits, secrets |
| [Stage 6: Delivery](/en/notes/stage-6-delivery/) | Tests, CI, and deployment | `httptest`, Docker, GitHub Actions |

## Note Status

Chinese is the source-of-truth learning language for detailed coaching notes. English pages provide stable summaries and navigation. When a Chinese note becomes a durable reference, add or update its English counterpart under `website/en/`.
