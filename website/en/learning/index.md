# Learning Path

This path assumes you already know JavaScript / Node.js, HTTP / REST, and basic DevOps. The focus is not memorizing syntax first, but moving quickly into Go's engineering model.

Core principles:

1. Every stage should produce something runnable.
2. Learn the standard library before adding frameworks.
3. Make error boundaries explicit before creating abstractions.
4. Let project needs decide what to learn next.

## Main Track

- Stage 1: startup, Go module, basic syntax, CLI, and tests.
- Stage 2: networking, HTTP, JSON, timeout, and `context`.
- Stage 3: concurrency, goroutine, channel, `errgroup`, and bounded parallelism.
- Stage 4: service, HTTP server, config, logs, and health checks.
- Stage 5: AI integration, LLM calls, retry, rate limiting, and structured output.
- Stage 6: delivery, tests, CI, Docker, and deployment.
