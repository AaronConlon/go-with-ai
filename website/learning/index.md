# 学习路径

这条路线默认你已经熟悉 JavaScript / Node.js、HTTP / REST 和基础 DevOps，因此不会把大量时间花在“什么是变量、函数和循环”上，而是尽快进入 Go 的工程用法。

核心原则：

1. 每个阶段都要有可运行成果。
2. 先掌握标准库，再评估第三方库。
3. 先写清楚错误边界，再追求抽象。
4. 用项目需求反推学习内容。

## 主线

- 阶段 1：启动期，掌握 Go module 和基本语法。
- 阶段 2：网络期，掌握 HTTP 和 JSON。
- 阶段 3：并发期，掌握 goroutine、channel、`context` 和 `errgroup`。
- 阶段 4：服务期，掌握 server、配置、日志和健康检查。
- 阶段 5：AI 整合期，掌握 LLM API 调用、重试、限流和结构化输出。
- 阶段 6：验证与部署期，掌握测试、CI 和部署。
